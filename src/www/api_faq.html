Marchesa Bot FAQ

This page is an FAQ on how to use Monarch Bot, possible problems and fixes; it will provide help for the Squire Bot integration.

Squire Bot Integration

This bot is integrated quite closely with Squire Bot, to enable trice bot in your Squire Bot tournament you need to set the tricebot-enabled property to true.
!update-properties "Tournament Name" tricebot-enabled=true

If you want to use player deck verification then you need to set the player-deck-verification property to true.
!update-properties "Tournament Name" player-deck-verification=true

The cockatrice-name command will automatically update the data that is sent to the trice bot on game creation. However the deck hashes cannot be updated after a game is started.

Player Deck Verification Issues

Players Being Kicked Wrongly

If a player is kicked the name that was sent to the API may be wrong it can be updated via the ../api/updateplayerinfo endpoint. When a player is kicked it will tell you the names it was expecting, which can then be used to find the player name that was typed in wrongly.

With Squire Bot...

With Squire Bot you can use the following commands: As a player you can use the cockatrice-name command, if this fails use the tricebot-update-player command. The syntax for the cockatrice-name command is:
!cockatrice-name New name
The syntax for the tricebot-update-player command is:
!tricebot-update-player "Tournament Name" "Match Number" "player name/mention" "new player name"
For example to change the name "Vldimir" to "Vladimir" in tournament "fnm-3/6", match 5, the command would be:
!tricebot-update-player fnm-3/6 5 Vldimir Vladimir

Alternatively You Can Disable Player Deck Verification...

Alternatively you can disable player deck verification with the ../api/disableplayerdeckverification endpoint.

With Squire Bot...

With Squire Bot you can use the command:
!ticebot-disable-pdi "Tournament Name" "Match Number"
For example the command to disable player deck verification for match 5 of tournament "fnm-3/6" the command would be:
!tricebot-disable-pdi fnm-3/6 5

Games Not Showing in Cockatrice

Wrong Cockatrice Version

If you have a cockatrice with version of 2.8.0 or less then the client does not support games made by spectators. Current Cockatrice version. The version in which support for games with no players was introduced is: 2.8.1 beta 2

Wrong Game Filter

You should have "Show password protected games" enabled; game type filters should all be un-ticked and; the spectator filters should also all be unticked. If these are your settings then it should show the games. You can use the "Creator Name" filter to find the game far faster, for Squire Bot the creator is called "Monarch Bot".

Replay Cannot Be Downloaded

If the replay cannot be downloaded with a file not found message then one of the following is the case:
  • The game has ended and some players are still in the game or;
  • The server is not responding to replay download requests or;
  • The server has not created a replay
If the game is still in progress then a webpage will appear showing which players are in the game.

Download All Replays For A Tournament

To download all the replays for a Squire Bot tournament the download-replays command can be used, which will return a ZIP file of the replays.
!download-replays "Tournament Name"
